# Разработка образовательной платформы для размещения онлайн-курсов с интеграцией Telegram-бота

Веб-приложение для создания и прохождения онлайн-курсов с поддержкой модулей, уроков, тестирования, отслеживания прогресса, приватных курсов и уведомлений через Telegram-бота.

## О проекте

Платформа предназначена для двух основных ролей:
- **преподаватель** — создаёт курсы, добавляет модули, уроки, учебные блоки и тесты, отслеживает прогресс обучающихся;
- **обучающийся** — записывается на курсы, изучает материалы, проходит тестирование, просматривает результаты и получает уведомления.

Дополнительно реализована интеграция с **Telegram-ботом**, который позволяет:
- привязать аккаунт по одноразовому коду;
- получать уведомления о новых уроках, тестах и дедлайнах;
- просматривать список курсов и ближайшие дедлайны.

## Основные возможности

### Для обучающихся
- регистрация, авторизация и восстановление пароля;
- просмотр каталога курсов;
- поиск, фильтрация и сортировка курсов;
- запись на публичные и приватные курсы;
- просмотр структуры курса: модули, уроки, вложения, учебные блоки;
- прохождение тестов;
- просмотр прошлых попыток и результатов;
- отслеживание процента прохождения курса;
- просмотр ближайших дедлайнов в профиле;
- получение уведомлений через Telegram.

### Для преподавателей
- создание и редактирование курсов;
- настройка приватного доступа по коду и паролю;
- добавление модулей и уроков;
- добавление учебных блоков разных типов: текст, изображение, GIF, аудио, видео;
- загрузка вложений к урокам;
- создание тестов и вопросов разных типов;
- просмотр прогресса обучающихся;
- просмотр результатов тестирования.

### Telegram-бот
- привязка аккаунта по одноразовому коду;
- команды `/start`, `/courses`, `/deadlines`, `/unlink`, `/help`;
- уведомления о новых уроках и тестах;
- напоминания о дедлайнах за **3 дня**, **24 часа** и **3 часа**.

## Структура проекта

```text
config/      # настройки проекта, маршрутизация, запуск приложения
courses/     # логика курсов, уроков, тестов, прогресса
users/       # пользователи, профиль, Telegram-интеграция
templates/   # HTML-шаблоны
static/      # CSS, JavaScript, статические файлы
media/       # загруженные пользователями файлы

```


## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <repository_url>
cd <repository_folder>
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
```

#### Windows
```bash
venv\Scripts\activate
```

#### Linux / macOS
```bash
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Создайтк файл `.env` и укажите свои параметры проекта.

```env
DB_NAME=
DB_USER=
DB_PASSWORD=
DB_HOST=
DB_PORT=

EMAIL_HOST=
EMAIL_PORT=465
EMAIL_USE_SSL=True
EMAIL_HOST_USER=
EMAIL_HOST_PASSWORD=
DEFAULT_FROM_EMAIL=

DJANGO_SECRET_KEY=

TELEGRAM_BOT_USERNAME=
TELEGRAM_LINK_SECRET=
TELEGRAM_BOT_TOKEN=
TELEGRAM_BOT_POLL_TIMEOUT=30
SITE_URL=http://127.0.0.1:8000
```

### 5. Применение миграций

```bash
python manage.py makemigrations
python manage.py migrate
```

### 6. Запуск веб-приложения

```bash
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000/
```

### 7. Запуск Telegram-бота

Бот запускается **отдельно** от веб-приложения.

Запуск выполняется в отдельном терминале:

```bash
python manage.py runtelegrambot
```


## Приватные курсы

В системе поддерживаются **приватные курсы**:
- приватный курс не отображается в общем каталоге;
- доступ к нему предоставляется по **коду** и **паролю**;
- код нормализуется, пароль хранится в защищённом виде;
- после успешного входа пользователю выдаётся доступ к содержимому курса.

## Тестирование

Подсистема тестирования поддерживает четыре типа вопросов:
- выбор одного ответа;
- выбор нескольких ответов;
- ввод текста;
- установление соответствия.

После прохождения теста система:
- сохраняет попытку пользователя;
- рассчитывает итоговый балл;
- показывает результат;
- позволяет просматривать прошлые попытки.

## Telegram-интеграция

Привязка аккаунта выполняется через **одноразовый код**, который генерируется в профиле пользователя. После успешной привязки в профиле сохраняются данные Telegram, а код становится недействительным.

Система уведомлений использует настройки профиля пользователя и отправляет сообщения:
- о новых уроках;
- о новых тестах;
- о приближающихся дедлайнах.

Для предотвращения повторной отправки одинаковых уведомлений используется журнал уже отправленных сообщений.
